### A star 算法浅析
* f(n)= g(n) + h(n) g(n)为目前节点到开始节点所需的步数，h(n)为开始节点到目标节点的估算距离，算法的优劣取决于h(n)
* [算法基本原理详见][1]

### canvas结合A*算法思路
* 基本工作
  
``` undefined
1， 把像素点根据屏幕大小 映射成一个二维数组, 一个数组的元素单位依步长为一个单位，每个元素按对象保存；
2， 随机生成的方形障碍物，按坐标和步长计算出面积区域，依次映射到二维数组上 值为1；
3， 存储start 和 starget 坐标，开始计算路径
4， 四方向检索并计算出f(n)后加入open数组，当前节点加入closed数组
5， 根据一定的算法（哈曼顿算法）计算出最小f(n)的节点，把当前节设为该节点的父节点，再把该节点设为当前节点；
6， 重复4-5步骤
7， 当估算距离h(n)===1的时候结束循环
8， 根据closed数组以及target的父节点追踪出完整路径path
9， 根据path渲染页面让角色自由移动
```
